Description

This is a self made/designed component based 2D game engine. (command input system, controller support, imgui support, text rendering, sprite animations, collisions and audio)

Gameobjects .h .cpp

The gameobjects exist to group components together, handle their lifetime, engine loop calls and give them information about the position, scene and other attached components/gameobjects. Gameobjects are final classes, this is so that functionality is only implemented through components.

Components .h .cpp

Components can be used for anything, the only requirement is that they are attached to a gameobject. A gameobject provides 2 templated functions to manage components AddComponent() and GetComponent(). The GameObject validates the component type at compile time using the IsComponent concept and, in most IDEs, the constraints are visible through IntelliSense, making it very user-friendly.

Transform .h .cpp

Every gameobject has a world and local position. Local position is the relative position to the parent gameobject, world position is the absolute position in the scene. These can only be accessed through public member functions. This is done to prevent unnecessary calculations, by setting a dirty flag whenever the positions change. When getting the world position, the engine can now know if the position needs to be recalculated depending on the flag.

Audio .h .cpp

By default the engine uses SDL_mixer for audio, but offers the possibility to implement and swap other libraries. This is done by using the service locator pattern, with this you can easily create your own SoundSystem and register it to the SoundSystemLocator. (here go a little indepth to explain what my current SDLSoundSystem does)

Input🔗

The input manager uses the command pattern to bind input events to actions. The engine supports both keyboard and multiple controllers controllers at the same time. By default only xbox controllers are supported, but by inherriting from the Controller class and using its interface you can technically use any input device you want.

Rendering🔗

The renderer is a singleton that provides a simple interface for rendering textures, masked textures and lines. These functions are used in RenderComponent and its derived components TextRenderer and SpriteRenderer.

ImGui🔗

The engine allows the use of ImGui through a ImGui render component. The only thing you need to do is use AddWindowFunction to pass a std::function that holds the ImGui window code. After that the engine will take care of the ImGui input and rendering.
