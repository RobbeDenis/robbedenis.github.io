Description

This is a self made/designed component based 2D game engine. (command input system, controller support, imgui support, text rendering, sprite animations, collisions and audio)

Gameobjects🔗

The Game objects only exist to group components together and handle their lifetime/ function calls. This engine focusses on composition, so all functionality can only be added through components. This is enforeced by making GameObject final.

Audio🔗

By default the engine uses SDL_mixer for audio, but offers the possibility to implement and swap other libraries. This is done by using the service locator pattern, with this you can easily create your own SoundSystem and register it to the SoundSystemLocator. (here go a little indepth to explain what my current SDLSoundSystem does)

Input🔗

The input manager uses the command pattern to bind input events to actions. The engine supports both keyboard and multiple controllers controllers at the same time. By default only xbox controllers are supported, but by inherriting from the Controller class and using its interface you can technically use any input device you want.

ImGui🔗

The engine allows the use of ImGui through a ImGui render component. The only thing you need to do is use AddWindowFunction to pass a std::function that holds the ImGui window code. After that the engine will take care of the ImGui input and rendering.
