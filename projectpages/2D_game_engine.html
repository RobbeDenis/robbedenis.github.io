2D Game Engine

This is a self made 2D game engine, written in c++, using SDL2 for rendering and SDL_mixer for audio. The engine uses a component based architecture, where functionality is implemented through components that are attached to gameobjects. Some other features of the engine are: Command based input system, ImGui support, State machines, Observers, Text and sprite rendering, Sprite animator and XBox 360 controller support. The Github repository also contains a demo game Pacman, that showcases the features of the engine.

Gameobjects
.h .cpp

The gameobjects exist to group components together, handle their lifetime, engine loop calls and give them information about the general state of the game and other objects. To enforce that functionality can only be added through components, gameobjects are made final.

Components
.h .cpp

Components can be used for anything, the only requirement is that they are attached to a gameobject. The only way to create a valid component is by using AddComponent() of a gameobject. The gameobject will then take care of initializing, updating and destroying the component.

Transform
.h .cpp

Every gameobject has a world and local position. To prevent updating every object's position every frame, the positions can only be accessed through getters and setters. By using a dirty flag, the position is only recalculated when needed.

Audio
.h .cpp

By default the engine uses SDL_mixer for audio, but offers the possibility to implement and swap to other libraries. This is done by using the service locator pattern, with this you can easily create your own sound system and register it to the SoundSystemLocator. Loading and playing audio gets done on a separate thread to prevent the main thread from freezing.

Input
.h .cpp

The input manager uses the command pattern to bind input events to actions. The engine supports both keyboard and multiple controllers at the same time, by default only xbox controllers are supported. Keyboard input is handled through SDL events, while controllers use XInput.

Rendering
.h .cpp

The renderer is a singleton providing a simple interface that uses SDL for rendering textures, masked textures and debug lines. These are mainly used in RenderComponent and its derived components TextRenderer and SpriteRenderer.

ImGui
.h .cpp

The engine allows the use of ImGui through an ImGui render component. The ImGui window code can be passed to the component using a std::function. After that, the engine will take care of the rendering and input.
