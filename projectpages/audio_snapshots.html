Metasound Audio Snapshots

Metasound Audio Snapshots is a plugin for Unreal Engine 5 that makes it possible to record and playback specific parts of a Metasound graph at runtime. This plugin was made to record and use audio data outside of the Metasound system and reduce the CPU usage by giving control over what and when audio generation can be replaced by its own 'cached' version.

Description

Metasound is Unreal Engine's system for real-time audio synthesis and processing. It provides a node-based interface for creating real-time complex audio. This plugin extends this system by allowing users to use cached versions of the audio as playback or send generated audio data to other parts of the engine.

The snapshot node can be triggered to start recording the audio for a specified duration or size. On completion, a signal is sent out and the snapshot is ready to be used. The node also makes use of the new configurable interface, allowing the user to control the inputs, outputs and some extra internal settings without the extra visual clutter that would normally be present.

Because the c++ side of metasounds is barely documented and not used by many people yet, I mainly looked at the source code to find the best practices to create this plugin.

Buffer snapping

To reduce artifacts on playback, the snapshot node offers two snap types: In Frequency and Last Zero Crossing. Both these options will resize the output buffer in its own way, this does not affect the internal recording buffer, only the output.

In Frequency will look at the given frequency input and resize the buffer to fit the maximum amount of periods of that frequency. This way, the output can be looped over seamlessly even without having to calculate the exact size of the buffer for every possible frequency needed.

Last Zero Crossing will look for the last zero crossing in the recorded buffer and cut the buffer there. The last sample will be set to 0 to ensure a clean cut.

Trigger precision

A snapshot node has an option to adjust its trigger precision or rate. There are two modes available: Sample Rate and Block Rate.

For very simple snapshots block rate would be enough, this means once start is triggered, the snapshot will start recording at the beginning of that block.

But in cases where the recording has to start on a very specific sample, sample rate can be used. This way the exact sample start is triggered, the recording will begin.

Metadata

In cases where the output buffer is resized because of snap types or downsampling, some metadata can be enabled to keep track of the original size. This can then be used to know how fast the output should be played without morphing the pitch.

There are some other metadata options available, but these will most likely be removed and replaced by a more generic system.
